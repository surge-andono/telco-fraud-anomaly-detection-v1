# 🧠 Detection Methodology: Isolation Forest

 location at src/detector.py

This project utilizes **Unsupervised Machine Learning** to detect anomalies without needing pre-labeled historical data.

### Why Isolation Forest?
Unlike clustering (K-Means) which finds groups, **Isolation Forest** focuses on isolating outliers. In Telco Fraud, anomalies are "few and different," making this algorithm highly effective.


### Technical Workflow:
1.  **Feature Scaling**: Using `StandardScaler` to ensure features like `daily_calls` (0-500) and `data_gb` (0-100) are on the same scale.
2.  **Partitioning**: The algorithm randomly selects a feature and a split value. Outliers (fraudsters) are isolated in fewer steps (shorter paths) than normal points.
3.  **Contamination Parameter**: Set at `0.02` (2%), assuming that true fraud is a rare event in the total subscriber base.
4.  **Anomaly Score**: Points with a score of `-1` are flagged for the Revenue Protection team to review.
